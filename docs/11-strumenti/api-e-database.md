# API e database

> API e database sono il cuore tecnico dei sistemi: l’Analista di Sistema deve comprenderli per analizzare integrazioni, dati e comportamenti.

---

## 🎯 Obiettivo della pagina

Comprendere:

- cosa sono API e database
- perché sono fondamentali per l’analista
- cosa è necessario sapere (e cosa no)
- come usarli nell’analisi

---

## 🧭 Cos’è un’API

Un’API (Application Programming Interface) è:

> **un’interfaccia che permette a sistemi diversi di comunicare tra loro**

---

### 📊 Esempio

- Sistema A chiede dati cliente  
- Sistema B risponde  

👉 tramite API

---

## 🧭 Cos’è un database

Un database è:

> **un sistema che memorizza e organizza dati**

---

### 📊 Esempio

- clienti  
- ordini  
- prodotti  

👉 dati strutturati e persistenti

---

## ⚖️ Principio chiave

> I sistemi comunicano tramite API e persistono dati nei database

---

## 🧩 Perché sono fondamentali

Per l’analista, API e database servono a:

- comprendere flussi  
- analizzare integrazioni  
- verificare dati  
- valutare impatti  

👉 Senza questa conoscenza, l’analisi è incompleta

---

## 🔍 Cosa deve sapere l’analista

---

### 🔗 API

- cosa espongono  
- quali dati scambiano  
- modalità (sincrona/asincrona)  
- gestione errori  

---

### 📦 Database

- entità principali  
- relazioni  
- struttura dati ad alto livello  

---

## ❌ Cosa NON serve sapere

- scrivere codice complesso  
- ottimizzazioni tecniche avanzate  
- dettagli implementativi  

👉 Serve comprensione, non sviluppo

---

## 🔄 Come si usano nell’analisi

---

### 📊 Analisi flussi

- quali API vengono chiamate  
- quali dati passano  

---

### 🔍 Analisi dati

- dove sono salvati  
- come sono collegati  

---

### ⚠️ Analisi impatti

- cosa cambia se modifico un dato  
- quali sistemi coinvolti  

---

## 📊 Esempio pratico

### Scenario
creazione ordine

### API
- chiamata per creare ordine  

### Database
- salvataggio ordine  

### Integrazione
- invio a sistema pagamenti  

👉 visione completa

---

## ⚠️ Problemi tipici

---

### Mancanza di comprensione API
→ errori integrazione  

---

### Dati incoerenti
→ problemi di sistema  

---

### Dipendenze non considerate
→ effetti a catena  

---

### Documentazione assente
→ difficoltà analisi  

---

## 🧠 Come lavora un analista senior

Un analista senior:

- comprende API a livello funzionale  
- interpreta modelli dati  
- identifica flussi e dipendenze  
- anticipa problemi  
- comunica con team tecnico  

👉 Non sviluppa, ma **capisce profondamente**

---

## 📌 Buone pratiche

- leggere documentazione API  
- comprendere modelli dati  
- verificare flussi reali  
- collaborare con sviluppatori  
- mantenere visione end-to-end  

---

## 🔄 Relazione con altri elementi

API e database collegano:

- architettura  
- integrazioni  
- flussi informativi  
- dipendenze  

👉 Sono il livello tecnico del sistema

---

## ⚖️ API vs database

| API | Database |
|-----|----------|
| comunicazione | persistenza |
| dinamico | statico |
| interazione | struttura |

---

## 🚀 In sintesi

API e database permettono di:

- comprendere come funzionano i sistemi  
- analizzare integrazioni  
- gestire dati  

E soprattutto:

> **collegare l’analisi funzionale alla realtà tecnica**
